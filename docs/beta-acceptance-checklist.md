# 無償β 受け入れチェックリスト

## 試してほしい流れ

1. Homeで「ここまでに残っていたこと」を見る
2. 「今日のことをひとつ残す」で、一言なし・一言ありの両方を試す
3. 「メモから見つける」に、日付あり行と日付なし行を貼り付ける
4. 日付なし行を「今日として残す」「日付なしで残す」「保存しない」に分けて確認する
5. 可能ならGoogleカレンダーから予定候補を確認する
6. 「未来の自分への一文」を1件残す
7. 「つらい時」を開き、無理なく見られる範囲で触る
8. Settingsで保存方針と無償βの範囲を見る
9. お問い合わせフォームを開く

## 見てほしい観点

- 「何もしていなかったわけではない」と感じられるか
- 保存前確認で、短いラベルやカテゴリを直しやすいか
- Googleを使わなくても、今日のことやメモ貼り付けから試せるか
- メモの日付なし行を勝手に今日扱いされないことが分かるか
- Homeに戻った時、「今回追加」「最近見つかった証拠」「カテゴリ/根拠行」が自然に読めるか
- 390px前後のスマートフォン幅で、横スクロールやボタン被りがないか
- 「つらい時」が強く励ましたり、無理に行動させたりしていないか

## フォームで聞きたいこと

- 何が残っていると見つけやすかったか
- どの画面で迷ったか
- 保存前確認の言葉は短く直しやすかったか
- Googleを使わない場合でも試せそうか
- 改善点、不具合、使ってみた感想、連絡事項がフォームで送りやすいか
- また必要な時に開きたいと思えたか
- 7月10日公開前に直した方がよい分かりにくさがあるか

## 任意: Google実予定保存の本人確認

Googleログインと同意は本人操作で行います。スクショや録画は残さないでください。

1. Googleカレンダーに、今日を含む過去7日以内で `RAW_DO_NOT_SAVE_0710 面接確認テスト` という予定を作る
2. 任意で場所に `LOCATION_DO_NOT_SAVE_0710`、詳細に `DETAIL_DO_NOT_SAVE_0710` を入れる
3. Vercel本番 `https://tsuzuribi-alpha.vercel.app/` を開く
4. Googleカレンダー画面でGoogleログイン/同意を行う
5. 実予定が読み込まれることを確認する
6. 保存前確認に進む
7. 短いラベルを `面接予定を確認した` のように編集する
8. 保存する
9. Homeに反映されることを確認する
10. landing演出が出ることを確認する
11. DevToolsでlocalStorageを確認し、raw情報が残っていないことを確認する

確認用スニペット:

```js
const saved = Object.values(localStorage).join("\n");
[
  "RAW_DO_NOT_SAVE_0710",
  "rawTitle",
  "eventId",
  "calendarId",
  "LOCATION_DO_NOT_SAVE_0710",
  "DETAIL_DO_NOT_SAVE_0710",
  "attendees",
  "description",
].map((key) => [key, saved.includes(key)]);
```

すべて `false` であれば、raw情報はlocalStorageに残っていません。
