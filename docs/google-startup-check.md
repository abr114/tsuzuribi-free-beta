# Google起動時確認の方式メモ

無償βでは、Googleカレンダー確認はユーザー操作による手動確認を基本にします。
アプリ起動時の自動確認、自動保存、裏側定期取得は実装しません。

## 無償βでやること

- ユーザーが `Google連携を確認する` を押した時だけ、Google認可へ進みます。
- ユーザーが読むカレンダーと期間を選びます。
- 取得した予定は保存前確認に出します。
- 保存するのは、ユーザーが確認した短いラベル、カテゴリ、日付だけです。
- Google予定タイトルの生データ、eventId、calendarId、location、attendees、descriptionは保存しません。

## 無償βでやらないこと

- 起動時のGoogle自動確認
- Google予定の自動保存
- refresh token保存
- 裏側定期取得
- DBへのGoogleトークン保存
- Plus向けの定期同期

## 将来の段階案

1. DBと認証を導入し、users と user_consents を整える。
2. Google連携の同意文面を、手動取得と起動時候補更新で分ける。
3. 起動時に候補更新する場合も、自動保存はせず、保存前確認に積むだけにする。
4. refresh token保存が必要な機能は、保存先、暗号化、失効、同意撤回を設計してから検討する。
5. 裏側定期取得はPlus以降の候補とし、無償βには入れない。

## UXメモ

起動時に候補更新を入れる場合も、画面上では「新しく見つかった候補があります」と表示するだけにします。
ユーザーが確認しない限り、つづりびの証拠箱には入りません。
